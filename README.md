# teledex

`teledex` 是一个通用的 `Telegram -> Codex` 控制桥接项目。

目标不是做复杂的平台托管系统，而是提供一个轻量、直接、可持续的使用方式：用户在 Telegram 里发消息，系统把消息路由到指定的 Codex 会话与工作目录中执行，并把过程预览与最终结果再回发到 Telegram。

## 项目定位

- 通过 Telegram 远程控制 Codex
- 支持会话持续，而不是每次重新开始
- 支持工作目录绑定，面向真实项目执行
- 支持过程预览，提升长任务可见性
- 保持通用、轻量、不绑定特定平台形态

## 第一版范围

第一版只聚焦最小可用能力：

- Telegram 收发消息
- 用户白名单
- 新建会话
- 继续当前会话
- 会话列表与切换
- 绑定工作目录
- 过程 `draft` 预览
- 最终结果回传
- 错误反馈
